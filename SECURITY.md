## 报告漏洞

**请勿在公开 GitHub Issue 中报告安全漏洞。**

请通过以下方式私下联系维护者（渠道待正式建立时可替换为专用邮箱或安全页面）：

- 维护者邮箱：**待补充**（请在 CONTRIBUTING 或项目主页更新后使用正式渠道）

请在报告中包含：

- 问题描述与影响范围
- 复现步骤或 PoC（若可行）
- 受影响版本

我们会在确认后尽快回复，并在修复发布前协调披露时间。

## 不可信输入（case / 网格 / MCP）

asimu 可能读取用户或 Agent 提供的算例与网格文件。在 `io` 层（及 MCP 沙箱）规划以下限制，详见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) §8.5.7、[DATA_MODEL.md](docs/DATA_MODEL.md) §13：

| 限制项 | 规划默认值 | 目的 |
|--------|------------|------|
| 单文件大小 | 256 MiB | 防止内存耗尽 |
| 最大单元数 | 10⁸ | 防止恶意超大网格声明 |
| case 文本行数 | 10⁶ | 防止解析 DoS |
| 路径 | 禁止 `..` 与工作区外路径 | 路径遍历 |
| 符号链接 | 默认不跟随（实现期定案） | 逃逸攻击 |

**原则**

- CLI、库、MCP **共用**同一 `io` 校验逻辑（Parse → Validate → Trust）
- 错误信息对用户可见时，避免泄露系统绝对路径（可配置）
- MCP **禁止** shell 类 Tool；见 [docs/MCP.md](docs/MCP.md)

## 依赖安全

CI 计划集成 `cargo audit` 进行依赖漏洞扫描；本地可运行：

```bash
cargo install cargo-audit
cargo audit
```

## 敏感配置

- 密钥、Token **不得**写入 case/TOML 或提交到仓库
- 仅通过环境变量注入（见 `AGENTS.md` 安全红线）
